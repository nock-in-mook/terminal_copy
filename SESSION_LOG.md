## ① 「発行元が不明」警告が毎回出る理由
`即ランチャー.exe` は `pythonw.exe` をコピーしてアイコンを付け替えた**未署名EXE**で、しかも **Gドライブ（Google Drive）上に置いてある**。Windowsはクラウド/ネットワークドライブ上の未署名EXEに対して毎回SmartScreen警告を出す仕様なので、これは正常な挙動。避けるには:
- ショートカット経由で起動する（`即ランチャー.lnk` はバッチが作ってくれる）
- またはEXE自体をローカル（`%LOCALAPPDATA%`）に置く

- 別のPCで `一発更新_即ランチャー.bat` を実行 → そのPCの `C:\Python314\` が `_pth` に焼き込まれる

- このPCで `即ランチャー.exe`（= pythonw.exeコピー）を起動すると、焼き込まれたパスの stdlib を読みに行く

## 方針

`_pth` は静的ファイルで動的解決できない。でも根本原因は「**ビルド成果物（exe/dll/_pth）をGoogle Driveに置いて複数PCで上書きし合ってる**」こと。これを解決すれば「発行元不明」警告も同時に消える（EXEがローカルドライブから起動するため）。

具体的には：
1. **ビルド先を `%LOCALAPPDATA%\即ランチャー\` に変更**（Gドライブじゃなくローカル）
2. `即ランチャー.exe` + `python3xx.dll` + `_pth` は各PCのローカルに置く
3. **ソースコード (`.pyw`) はGドライブのまま** — 編集は従来通り即反映
4. ショートカットの TargetPath = ローカルEXE、Arguments = GドライブのPYW
5. Gドライブ側の古いビルド成果物は削除
6. `.gitignore` に該当パスを追加

これで各PCで `一発更新_即ランチャー.bat` を1回走らせれば、そのPC専用のexe/dll/_pthがローカルに作られて、他PCに干渉しない。

## 他PCでも `一発更新_即ランチャー.bat` を走らせるべき理由

1. **そのPCでもEXEがまだGドライブ上にある可能性**
   - 今回の修正でGドライブ上の `即ランチャー.exe` / `python3xx.dll` / `_pth` を削除してコミット＆プッシュした
   - 他PCで `git pull` すると、Gドライブ上のこれらファイルが**消える**
   - → 他PCの現状のショートカットがGドライブのEXEを指してるなら、**起動できなくなる**
   - → `一発更新_即ランチャー.bat` を走らせて、ローカル(`%LOCALAPPDATA%\即ランチャー\`)に再ビルドする必要がある

2. **ショートカットも更新される**
   - `_setup_shortcuts.py` がショートカットをローカルEXEを指す形で作り直す

3. **Python 3.14が無ければ自動で入れてくれる**

## 手順（他PCで）

```
1. git pull（Gドライブ上の変更を反映）
2. 一発更新_即ランチャー.bat をダブルクリック
```

それだけ。5〜10秒で終わる（Python 3.14が既にあれば）。無ければwingetインストール分で数分。

**ただし注意**: `git pull` しなくてもGoogle Drive同期で `.bat` や `.pyw` は勝手に最新化される。でもEXE削除だけはGoogle Driveの同期でも反映されるから、結局**どのPCでも一度はbatを走らせる必要がある**。じゃないとEXEが消えたままになる。
